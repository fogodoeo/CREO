# 크레와트 시네마틱 선택 장면 제작 지침 v3

## 1. 핵심 구조

각 문항은 가로 3:2 삽화 3장을 사용한다.

1. `neutral`: 선택 전, 왼쪽과 오른쪽 가능성을 한 화면에 함께 보여준다.
2. `left`: 왼쪽 선택 후, 왼쪽 결과만 화면 전체에 보여준다.
3. `right`: 오른쪽 선택 후, 오른쪽 결과만 화면 전체에 보여준다.

선택하지 않은 결과를 같은 이미지에 넣거나 검은 안개로 가리지 않는다. 선택 후에는 고른 결과 전용 이미지가 장면 전체를 채운다. 기본 자산 수는 `12문항 × 3장 = 36장`이다.

## 2. 선택 전 `neutral` 이미지

- 왼쪽과 오른쪽 선택의 핵심 차이가 한눈에 보여야 한다.
- 좌우의 크기, 밝기, 정보량을 비슷하게 맞춰 어느 쪽도 정답처럼 보이지 않게 한다.
- 한쪽의 핵심 대상은 1~2개로 제한한다.
- 필요하면 중앙에 고민하는 정상적인 성인 인물을 장면 일부로 그릴 수 있다.
- 별도 레이어로 올리는 백지 인형, 흰색 마네킹, 공통 전경 아바타는 사용하지 않는다.
- 중앙 인물이 없어도 두 가능성이 충분히 구분되면 사물과 공간만 사용한다.
- 두꺼운 카드 테두리나 강한 분할선보다 공간, 거리, 조명과 옅은 연필선으로 좌우를 구분한다.

## 3. 선택 후 `left`와 `right` 이미지

- 결과 이미지는 선택한 상황 하나만 담는다.
- 선택한 대상과 행동을 더 가까운 시점으로 보여주되 핵심 피사체가 잘리지 않게 한다.
- 반대 선택의 인물, 대상, 장면을 남기지 않는다.
- 검은 안개, 검은 막, 분할선과 화면 절반 가리기를 이미지나 CSS에 넣지 않는다.
- 인물이 필요하면 모의고사 삽화풍의 정상적인 사람을 이미지 안에 직접 그린다.
- `neutral`과 같은 공간, 종이 질감, 카메라 높이와 명암을 유지한다.

## 4. 공통 이미지 스타일

모든 프롬프트에 다음 조건을 포함한다.

```text
현대의 현실적인 크레스티드게코 사육 또는 행사 공간.
한국 모의고사 문제지의 세련된 흑백 삽화풍.
섬세한 연필·잉크 선과 절제된 회색 명암, 따뜻한 아이보리 종이 질감.
가로 3:2, 자연스러운 원근과 하나의 수평선.
글자, 숫자, 로고, 휴대폰, 화면, 사진, 기록지와 판타지 요소를 넣지 말 것.
백지 인형, 흰색 마네킹, 별도 전경 아바타를 넣지 말 것.
사람이 필요하면 현실적인 성인 인물을 장면 안에 자연스럽게 통합할 것.
크레스티드게코는 실제 Correlophus ciliatus의 머리, 볏, 큰 눈, 발가락 패드와 꼬리를 정확히 표현할 것.
과도한 소품과 설명용 오브젝트를 넣지 말 것.
```

## 5. 인물과 크레 규정

- 인물은 정상적인 얼굴, 손, 관절과 옷을 가진 성인으로 그린다.
- 과장된 고민 포즈, 만화식 물음표와 성별을 과장하는 장식은 피한다.
- 한 문항의 세 이미지에서 인물의 외형과 공간이 자연스럽게 이어져야 한다.
- 크레는 넓은 삼각형 머리, 큰 눈, 눈 위에서 등으로 이어지는 볏, 네 다리, 접착 발가락과 잡는꼬리를 갖춘다.
- 추가 다리, 중복 꼬리, 비정상적인 발가락, 다른 도마뱀 종 형태가 보이면 폐기한다.

## 6. 금지 요소와 단순화

- 휴대폰, 태블릿, 노트북, 모니터
- 사진, 엽서, 기록지, 체크리스트, 달력
- 읽을 수 있는 글자, 숫자, 로고와 화면 속 화면
- 별도 합성용 백지 인형 또는 마네킹 캐릭터
- 검은 안개로 반대 결과를 숨기는 통합 결과 이미지
- 작은 물건을 잡거나 조작하는 복잡한 손
- 마법학교, 로브, 고딕 건축과 판타지 장식
- 설명용 오브젝트가 세 개 이상 쌓인 장면

정보는 물건의 개수보다 인물의 위치, 시선, 거리, 공간의 개방감과 상호작용으로 표현한다.

## 7. 웹 전환 규칙

- 기본 상태에서는 `neutral`만 보인다.
- 왼쪽 선택 시 `neutral`을 숨기고 `left`를 전체 프레임에 교차 전환한다.
- 오른쪽 선택 시 `neutral`을 숨기고 `right`를 전체 프레임에 교차 전환한다.
- 결과 이미지는 `object-fit: contain`으로 3:2 프레임 전체에 표시한다.
- 전환은 약 280~500ms, 다음 문항 이동까지 전체 약 900ms를 권장한다.
- 모션 감소 환경에서는 확대 없이 짧은 불투명도 전환만 사용한다.
- 이미지 위에 별도 인물, 안개, 가림막과 구분선을 합성하지 않는다.

## 8. 선택지 좌우 무작위화

문장 순서를 무작위화할 때는 이미지 경로도 함께 교환한다.

```js
const leftScene = question.visualFlipped ? scene.right : scene.left;
const rightScene = question.visualFlipped ? scene.left : scene.right;
```

버튼의 실제 왼쪽/오른쪽 위치와 결과 이미지가 항상 일치해야 한다. 전체 이미지를 CSS로 좌우 반전하는 경우 인물과 공간 방향이 어색해질 수 있으므로 반드시 육안 검수한다.

## 9. 성능과 접근성

- 최종 파일은 WebP 또는 AVIF로 저장한다.
- 1536×1024 기준 장당 250~550KB를 목표로 한다.
- 현재 문항의 3장을 우선 불러오고 다음 문항의 `neutral`을 미리 불러온다.
- 삽화 세트가 바뀌면 CSS 또는 자산 캐시 키를 변경한다.
- 삽화 없이 선택지 문장만 읽어도 답할 수 있어야 한다.
- 선택 전 이미지만 상황을 설명하는 대체 텍스트를 제공하고 결과 이미지는 장식 요소로 처리한다.
- 선택 상태는 색뿐 아니라 `aria-pressed`로도 전달한다.

## 10. 문항별 제작 절차

1. 두 선택의 차이를 한 문장으로 정의한다.
2. 좌우의 핵심 대상 또는 관계를 하나씩 정한다.
3. 두 가능성을 함께 담은 `neutral` 이미지를 제작한다.
4. 왼쪽 선택만 담은 `left` 이미지를 제작한다.
5. 오른쪽 선택만 담은 `right` 이미지를 제작한다.
6. 세 이미지의 공간, 스타일과 크레 형태를 확대 검수한다.
7. 좌우 선택지 무작위화 뒤에도 문장과 결과가 일치하는지 확인한다.
8. 아이폰을 포함한 터치 기기에서 이전 선택 상태가 다음 문항에 남지 않는지 확인한다.
9. 320px, 375px, 390px, 768px와 1280px 이상에서 검수한다.
10. WebP 압축과 캐시 키를 적용한다.

## 11. 최종 검수표

- [ ] 선택 전 이미지 한 장에서 왼쪽과 오른쪽 가능성이 즉시 구분된다.
- [ ] 선택 후에는 고른 결과 전용 이미지가 화면 전체에 나온다.
- [ ] 반대 결과, 검은 안개와 분할선이 결과 이미지에 남지 않는다.
- [ ] 백지 인형이나 별도 전경 아바타가 없다.
- [ ] 필요한 인물은 모의고사 삽화풍의 정상적인 사람으로 장면 안에 통합되어 있다.
- [ ] 좌우의 밝기와 정보량이 편향되지 않는다.
- [ ] 실제 크레스티드게코로 보이며 신체 오류가 없다.
- [ ] 문장 무작위화 후에도 좌우 결과 매핑이 정확하다.
- [ ] 아이폰에서 다음 문항이 선택된 상태로 시작하지 않는다.
- [ ] 모바일에서 가로 넘침과 핵심 피사체 잘림이 없다.
- [ ] 이미지 압축과 캐시 키 반영이 끝났다.

## 12. 완료 기준

한 문항은 `neutral`, `left`, `right` 세 이미지 검수, 좌우 매핑, 모바일·데스크톱·모션 감소 환경 확인, 이미지 압축과 캐시 반영까지 끝나야 완료로 본다.
