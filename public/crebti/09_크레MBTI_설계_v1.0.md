# 크레 MBTI 설계 v1.0

## 한 문장 목표

평소 MBTI와 크레를 대할 때의 MBTI를 바로 비교하고, 사용자가 `내가 뭘 골랐길래?`라고 궁금해지는 순간 BAND 상세 분석으로 자연스럽게 이어간다.

## 최종 흐름

1. 첫 화면에서 `테스트 시작`을 누르면 바로 첫 질문으로 이동한다.
2. 크레 사육·선택 상황 20개에 답한다.
3. 평소 MBTI를 고르거나 `잘 모르겠어요`를 누른다.
4. 평소 MBTI와 크레 MBTI, 달라진 축, 전체 응답속도를 즉시 확인한다.
5. 문항별 선택 근거·응답시간·기숙사는 BAND 가입자에게 공개한다.
6. 기숙사는 MBTI 결과와 분리해 현재 인원이 가장 적은 기숙사 중 답변 친화도가 높은 곳으로 배정한다.

## 이번에 추가한 것

- 축별 5개, 총 20개의 구체적인 상황형 질문
- 한 사람에게 보이는 선택지 위치를 정확히 10:10으로 섞는 위치 균형
- 탭이 보이는 동안만 측정하는 문항별 응답시간
- 0.4초 미만 연속 터치와 30초 초과 자리 비움 제외
- 평균 대신 개인 중앙값을 기본 속도로 사용
- 초기 유효 응답 10명 전에는 `기준 데이터 N/10` 표시
- 10명부터 현재 표본 평균과 빠른 순위 구간 표시
- 익명 결과를 OAuth 뒤 동일 세션으로 갱신하는 세션 ID
- 새 설문 버전 응답만 비교하는 통계 분리
- 질문의 상황만 보여주는 흰 종이·연필화 중립 삽화 1장과 문장형 선택지
- 실제 세부 결과를 블러로 미리 보여주고 BAND 가입 확인 시 해제하는 잠금
- 테스트 중에는 BAND 동선을 숨기고 첫 화면 보조 연결과 결과 잠금 해제에만 노출
- 결과 유형을 반복하는 카드 대신 `평소 MBTI → 크레 MBTI` 공유용 포스터 한 장
- 결과 직후 카카오·인스타그램·BAND 동일 크기 아이콘, 변화 설명과 속도는 두 개의 보조 구역으로 축약
- `다시 테스트하기`는 세부 결과 아래 최하단의 큰 버튼으로 배치
- 운영체제 설정을 따르는 라이트·다크 모드와 동적 브라우저 테마 색상

## 이번에 없앤 것

- 기숙사를 성격 테스트의 주 결과로 보여주는 구조
- 선택지마다 서로 다른 삽화를 붙여 이미지를 고르게 만드는 구조
- 흰색 이미지 카드, 반복되는 둥근 통계 카드와 떠 있는 BAND 버튼
- A/B 글자 노출
- 길고 현학적인 기숙사 설명
- 질문 선택 때마다 서버에 저장하는 방식
- 이전 12문항 결과와 새 20문항 결과를 같은 표본으로 계산하는 방식

## 문항 구성

| 축 | 측정하는 장면 | 문항 수 |
|---|---|---:|
| E/I | 대화·공유로 정리 / 혼자 관찰·기록으로 정리 | 5 |
| S/N | 현재 확인 정보 / 성장 흐름과 가능성 | 5 |
| T/F | 조건과 근거 / 취향과 관계 | 5 |
| J/P | 미리 정한 운영 / 반응에 맞춘 조정 | 5 |

각 선택지는 어느 한쪽이 더 좋은 사육자처럼 보이지 않도록 작성한다. 건강·안전 지식 정답을 묻지 않고, 두 방식 모두 가능한 상황만 사용한다.

## 파일럿 10명 이후 확인할 것

- 질문별 첫 번째 위치 선택률이 65% 이상인지
- 특정 문항 응답 중앙값이 전체의 두 배 이상인지
- 축별 5:0 결과가 지나치게 많은지
- `평소 MBTI → 크레 MBTI` 변화가 한 방향으로 과도하게 몰리는지
- 중도 이탈이 주로 몇 번째 문항에서 발생하는지
- 상세 분석 잠금 버튼 클릭률과 BAND 가입 확인률

표본 10명은 문구와 속도 이상을 찾는 초기 기준이다. 백분위 문구는 재미 요소로 사용하고 심리검사 정확도를 주장하지 않는다.
